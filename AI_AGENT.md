## 1. 개요

### 무엇을 만드는가
"고정자산 업무 어시스턴트". 사용자가 자연어로 자산·장애·계약·점검 관련 질문을 던지면, **단순 조회가 아니라** 분석 → 진단 → 조치안까지 단계적으로 제시하는 **채팅형 화면**.

### 시연 목표
- 좌측 "추천 질문" 클릭 → 중앙 채팅 영역에 사용자 질문이 떠오르고, AI Agent가 단계별로 답변
- 우측 패널에 "AGENT 분석 단계" 6단계가 차례로 채워지며 분석 진행 시각화
- 분석 종료 시 "관련 자산", "영향 비용", "추천 조치"가 자동 채워짐

### 핵심 차별 포인트 (시연 멘트용)
1. 자연어 질의 → 자산 데이터 자동 조회
2. 분석 6단계가 우측에 가시화 (의사결정 신뢰성↑)
3. 단순 답변이 아닌 **조치안(교체/임대/유지 비용 비교)** 제시
4. 자산이력·ITSM·전자결재로 한 번에 연결되는 액션 버튼

```

💰 이번 분기 교체 예산 예상해줘
📅 다음달 점검 일정 보여줘
📝 노후 장비 현황 보고서 작성해줘
```

> **고정 규칙**: 추천 질문 6개 중 **1·2번은 시연 시나리오 1·2번 질의와 완전히 동일한 텍스트**로 둔다. 시연자가 타이핑 없이 클릭만으로 두 시나리오를 진행할 수 있도록 한다. 3~6번은 더미(시연 중 클릭 안 함).

> **클릭 시 동작**: 항목 클릭 → 입력창에 텍스트 채우지 않고 **즉시 `askAgent(질문 텍스트)` 호출** (자동 전송). 사용자 버블이 채팅에 추가되고 분석이 바로 시작된다. 기존 1-④ 타이밍 시퀀스 그대로 적용.

- 초기 렌더 함수 `renderAiAgent()`:

  - 채팅 본문에 인사 메시지 1건 미리 표시
  - 6단계 분석 카드 초기 렌더 (모두 비활성)
- 동작 함수 (시연용 더미 시뮬레이션)
  - `askAgent(question)`: 사용자 버블 추가 → 300ms 간격으로 step 1~6 활성화 → 마지막에 에이전트 답변 + 관련 자산/비용/조치 카드 채움

- 좌측 항목 클릭 시 → 해당 텍스트를 input에 채우고 자동 전송하거나 즉시 `askAgent` 호출

## 7. 예외 상황 / 결정 사항

1. **동일 질문 반복**: 마지막 답변이 그대로 다시 표시되어도 무방 (시연이므로). 단계 카드는 매번 초기화 후 다시 활성화.

4. **사이드바 접힘 상태**: 본 화면은 영향 없음 (이미 적용된 `.sidebar.collapsed` 룰 그대로 동작).
5. **새 분석 시작 버튼**: 채팅 초기화 + 우측 카드 초기 더미로 복원. 좌측 추천 질문 클릭 횟수와 무관.

### 0. 화면 진입 애니메이션 (모든 시나리오 공통)

대시보드 화면과 동일한 패턴(`dashFadeUp` 키프레임 재사용 + `#view-ai-agent.active.anim-on` 선택자). `showSection('ai-agent')`에서 `triggerAiAgentAnimations()`를 호출하여 매 진입마다 재생.

#### 카드 등장 시퀀스 (총 약 0.95s)
| 영역 | delay | duration |
|---|---|---|
| 페이지 헤더 (`AI Agent` 제목 + 우측 버튼 2개) | 0s | .42s |
| 좌측 추천 질문 카드 | .08s | .42s |
| 좌측 즐겨찾는 분석 카드 | .16s | .42s |
| 중앙 채팅 카드 (헤더·본문·입력창 묶음, 본문은 빈 상태) | .20s | .42s |
| 우측 AGENT 분석 단계 카드 | .28s | .42s |
| 우측 관련 자산 카드 | .36s | .42s |
| 우측 영향 비용 카드 | .44s | .42s |
| 우측 추천 조치 카드 | .52s | .42s |

#### 항목 stagger
- 추천 질문 6개: `.20s` ~ `.50s` (60ms 간격)
- 분석 단계 6개: `.36s` ~ `.56s` (40ms 간격)

#### 인사 메시지 등장
- 채팅 본문은 진입 시점에 **비어 있음** (`renderAgentInit`이 빈 상태로 둠)
- `triggerAiAgentAnimations`가 우측 패널 등장 직후(`.55s`)에 인사 메시지 1건을 fade-in+pop으로 등장
- 클래스: `.msg.agent.greeting.anim-pop`, 키프레임: `aiBubblePop` (opacity 0→1 + translateY 8px→0 + scale .96→1.02→1)
- 이미 진행 중인 분석(사용자 버블/답변 카드/분석중)이 있으면 인사 메시지 건너뜀 (중복 방지)

#### 재생 트리거 — `triggerAiAgentAnimations()` 의사코드
```js
function triggerAiAgentAnimations() {
  var view = document.getElementById('view-ai-agent');
  var body = document.getElementById('ai-chat-body');
  var hasOngoing = body.querySelector('.msg.user') ||
                   body.querySelector('.msg.agent.answer') ||
                   body.querySelector('.msg.thinking');
  if (!hasOngoing) {
    body.innerHTML = '';
    setTimeout(_appendGreetingBubble, 550);
  }
  view.classList.remove('anim-on');
  void view.offsetWidth;          // reflow → 키프레임 재시작
  view.classList.add('anim-on');
}
```

#### "+ 새 분석 시작" 클릭 시
- `resetAgent()` → `renderAgentInit()` → 본문 빈 상태로 두고 `setTimeout(_appendGreetingBubble, 200)`으로 인사 메시지가 약간 빠르게 (.2s) 재등장. 카드 등장은 재생되지 않음 (이미 보이는 상태).

### 예외 / 결정 보류 사항

1. **상세 표시 방식 — 모달 vs 우측 패널**: 영상 자료 확인 후 확정. 우측 패널이 기존 채팅 화면의 "AGENT 분석 단계" 영역과 공간 충돌하므로, **목록 결과가 나오는 동안에는 우측 단계 카드를 접고 상세 패널이 슬라이드 인** 하는 방식이 자연스러움 (잠정 선호).
2. **"띠리리" 효과음**: 시연 환경(브라우저 무음 정책)을 고려해 **시각 연출(점멸·점 3개·진행바)을 기본으로 하고, 효과음은 옵션 토글**로 제공.
3. **목록이 길 경우**: 채팅 본문이 길어져 스크롤 부담이 큼 → 목록은 **상위 5건 + "전체 보기 → 원장관리 이동" 링크** 형태로 끊는다.
