## 3. 〔1.2~1.6 = 개발사 파트〕 핵심 정리 (p18~57)

### 1.2 실시간 자산 관리 (AI 관제·예측) — p18~24
- **1.2.1 (5p):** **EXAONE(LLM) + LightGBM(예측) 하이브리드 AI Agent**로 자산 관제·예산 예측 체계 구축.
  - 데이터 정제·표준화 5단계(진단→표준기준→정제·변환→품질검증·승인→지속관리)
  - 분석 학습 데이터셋 구축, **지식베이스/RAG 구축**(비정형 문서→Vector DB→EXAONE 연계, 환각 최소화)
- **1.2.2 (2p):** EXAONE 질의응답으로 자산 데이터 추출, 자산 상태 급변 시 긴급 보고·알림

### 1.3 예산 및 회계결산 고도화 — p25~34
- **1.3.1 (4p):** 예산 프로세스 개선 → **'자산관리 종합현황판'** 구축 + 예산 공통 모듈화(편성/배정/조정/환수/집행/재배정). 실시간 트래킹.
- **1.3.2 (3p):** **불용 예산 사전 탐지·적기 재분배**(LightGBM 회귀 + EXAONE 분류 Feature). 업무 개선 효과 표.
- **1.3.3 (3p):** 외부 감사 대응자료(주석 공시·연결 결산) 자동 검증 + 디지털 이력 관리(AS-IS 수작업 → TO-BE 자동).

### 1.4 자산별 프로세스(신청·검증) 개선 — p35~40
- **1.4.1 (3p):** 신청→검증→결재→승인→사후관리 단계 통합. **업무용 차량·사택·합숙소** 수기처리 표준화. 자산별 프로세스 현황판.
- **1.4.2 (3p):** **외부 시세 API·지도 API·인사DB 연계 자동 검증**.
  - 시세조사(한국부동산원 API), 출퇴근 거리·요건(지도 API), 면적 기준(인사DB 부양가족 수), 금액 한도(시세×면적×지역비율)
  - AS-IS(담당자 수기 조회·캡처) → TO-BE(자동 조회·전자문서 자동 반영)

### 1.5 데이터 연동 및 최신화 — p41~54
- **1.5.1 (4p):** 외부 공공/민간 데이터 연계(한국부동산원 전세가/시세 API, 네이버·카카오 지도 API, KB부동산, 국토교통부, 행안부 주소 API). **전체 시스템 아키텍처**(p44): 취득→감가상각→처분, 물품/재물조사/불용자산, 운영관리/건설중 자산/계약대장/임대관리, 국고보조금/감가상각 시뮬레이션. 연계: GSE(실시간 전문)·MCA(대외채널)·MFT(파일 송수신).
- **1.5.2 (7p):** **자산 전 Life-Cycle 관리** — 취득/운영/임대/유지보수/검사/권리관계/보험/계약서/도면을 자산별 통합 등재. 단계별 화면 예시 다수.
- **1.5.3 (2p):** **자산별 디지털 이력서** 생성. 신규 취득 시 취득원가·구성요소·보조금 자동 계산.
- **1.5.4 (1p):** 사용자 정의형 데이터 추출(셀프 서비스 쿼리).

### 1.6 업무지원 프로세스 — p55~57
- **1.6.1 (2p):** 표준화·매뉴얼화·시스템 제어. 계약(사무) 프로세스 도식화 + 단계별 점검사항.
- **1.6.2 (1p):** 단계별 필수 이행 체크리스트(수요발생→입찰공고→제안서접수→평가/낙찰→계약체결) + 단계별 템플릿. 미이행 시 다음 단계 진입 차단.

## 5. As-Is → To-Be 매핑 & 설계 방향성 메모

> ※ 아래는 "정리" 차원의 방향성 메모입니다. **실제 설계·구현은 별도 요청 시** 진행합니다.

### 5-1. 현행(As-Is) ↔ 제안(To-Be) 대응
| 현행(As-Is) | 제안(To-Be) | 제안서 근거 |
|------------|------------|-----------|
| 빽빽한 폼 기반, 메뉴 11개 | **카드형 모듈 대시보드 + 직관적 UX** | 1.1.1, 1.1.3 |
| 일반자산 취득/처분/이관 (수기 폼) | 신청→검증→결재→승인→사후관리 **단계 현황판** | 1.1.2, 1.4.1 |
| 부동산·차량 리스 계약(수기) | 외부 API·지도·인사DB **자동 검증** | 1.4.2 |
| 현황관리 조회(텍스트 그리드) | **GIS 지도 관제 + 시각화 리포트** | 1.1.1(p9·11), 1.1.4 |
| 결산/회계서식(수작업 마감) | **외부 감사자료 자동 검증·디지털 이력** | 1.3.3 |

### 5-2. 우리 프로토타입(데모)과의 정합성 점검
이미 데모로 구현된 것 ↔ 제안서 항목:
- ✅ 통합 대시보드(KPI·차트) ↔ p7·p11
- ✅ 지역별 관리 현황 지도 ↔ p9 (단, 제안서는 **서울 구 단위** 상세 — 우리는 전국 5개 권역)
- ✅ 내 업무 현황 팝업 ↔ p8 개인화
- ✅ AI Agent 채팅/시나리오 ↔ 1.2 AI Agent·RAG 질의응답 / p11 사이드바 'AI Agent'
- ✅ 보고서 화면 ↔ 1.3.3 감사자료 / 1.1.1 리포트
- ⬜ (미반영) 생애주기 모니터링(p10), 자산 신청→승인 단계 현황판(1.4.1), 예산 종합현황판(1.3.1), 불용예산 AI 탐지(1.3.2), 디지털 이력서(1.5.3)

### 5-3. 설계 시 고려할 방향성(메모)
1. **사이드바 11개 메뉴는 제안서 p11이 단일 기준** — 우리 데모 사이드바와 일치 유지.
2. **"단계 현황판" 패턴이 제안서 전반의 핵심 UX** (1.1.2·1.3.1·1.4.1·1.6) — 신청·승인·결산·계약을 모두 "단계별 진행 + 점검 체크리스트"로 표현. 데모 확장 시 1순위 후보.
3. **외부 API 자동 검증(1.4.2)·GIS(p9)** 는 "시연 시 임팩트 큰" 차별화 포인트 — 데이터는 데모용 가짜로 흉내 가능.
4. **AI(1.2)** 는 별도 분석한 `AI_AGENT_제안분석.md`(우리은행 AI에이전트 3계층)와 연결해 설계 — 멀티에이전트·신뢰도·RAG 컨셉.
5. As-Is의 빽빽함과 대비되는 "**Before/After**" 시연 구도가 설득력 높음(제안서도 AS-IS/TO-BE 대비를 반복 사용).

### 7-1. NH 시스템의 유형 분류

글로벌 고정자산관리 시스템은 두 계열로 나뉩니다:

| 유형 | 핵심 기능 | 대표 제품 | NH 해당 여부 |
|------|---------|---------|-----------|
| **회계/세무 중심형** | 감가상각·처분·결산·ERP 연동·세무신고 | Sage, NetSuite, Acumatica, Ecount | ✅ **NH 해당** |
| **운영/설비 중심형** | 위치추적·작업지시·유지보수·CMMS | IBM Maximo | ❌ NH 주범위 외 |

**NH는 금융회사이므로 "회계/세무 중심형"이 주축입니다.** 제안서도 감가상각·결산 자동화·외부 감사자료(1.3)·디지털 이력서(1.5.3)·ERP 연계(1.5.1 GSE/MCA/MFT)를 핵심으로 제시하고 있어 글로벌 방향과 일치합니다.
