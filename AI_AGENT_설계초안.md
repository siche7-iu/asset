## 1. 설계 개념 프레임

### 1-1. AI 에이전트 관리의 6축 (Genspark 채택)

모든 AI 에이전트 관리 제품에서 반복되는 6개 핵심 축. 이 콘솔의 각 화면은 이 6축 중 하나에 대응한다.

```
┌────────────┬────────────┬────────────┐
│  ① 빌드    │  ② 관찰    │  ③ 평가    │
│  에이전트   │  실행 경로  │  품질 점수  │
│  설계·구성 │  추적·모니터│  신뢰도 측정│
├────────────┼────────────┼────────────┤
│  ④ 비용    │  ⑤ 보안    │  ⑥ 人介入  │
│  사용량 파악│  권한·감사  │  사람이 결재│
│  (요약만)  │  정책·차단  │  승인 게이트│
└────────────┴────────────┴────────────┘
```

> ④ 비용은 모니터링 화면에 KPI 카드 1개로 요약. 독립 화면 불필요 (데모 임팩트 낮음).

### 1-2. 3종 사용자 유형 (Genspark 채택)

이 콘솔은 동시에 3종류 사용자를 만족해야 한다.

| 사용자 유형 | 역할 | 주로 보는 화면 | 관심 지표 |
|-----------|------|-------------|---------|
| **현업 담당자** | 자산관리, 부서담당자 | AI Copilot · 이상탐지 · 결재함 | 답변 정확도, 알림 수신, 승인 대기 |
| **AI 운영자** | 시스템 운영, AI 담당 | 빌더 · 모니터링 · 지식베이스 · 평가 | 오류율, 응답시간, 지식 최신성 |
| **관리자/감사** | 정보보안, 컴플라이언스 | 거버넌스 | 감사로그, 차단 현황, 권한 준수 |

### 1-3. 2개 레이어 구조 (우리 초안 유지)

```
┌──────────────────────────────────────────────────────────┐
│  사용자 레이어 (담당자·부서장이 매일 사용)                  │
│  AI Copilot · 이상탐지 보드 · 자동 리포트 · AI 결재함      │
├──────────────────────────────────────────────────────────┤
│  관리 레이어 (AI 운영자·시스템 관리자 사용)                 │
│  에이전트 빌더 · 모니터링 · 지식베이스 · 평가·품질 · 거버넌스│
└──────────────────────────────────────────────────────────┘
```

### 1-4. 설계 불변 원칙 (우리 초안 유지)

1. **설명가능성 우선** — 모든 AI 답변에 근거 자산(출처 데이터 링크)과 추론 단계 표시.
2. **사람이 최종 결재** — AI는 제안·초안까지. 실제 반영은 담당자 확인 필수.
3. **금융 컴플라이언스** — 전수 감사 로그, 역할별 접근 제어, 망분리 환경 전제.

## 2. 최종 메뉴 구조(IA)

### 최종 확정 메뉴 (총 9개 화면)

```
사이드바
├── [AI 에이전트]  ← 그룹 헤더 (사용자 레이어)
│   ├── AI Copilot          #/ai-copilot     ← 기존 #/ai-agent 개선
│   ├── 이상탐지 보드        #/ai-anomaly     ← 신규
│   ├── 자동 리포트          #/ai-report      ← 신규
│   └── AI 결재함            #/ai-approval    ← 신규 (Genspark 승인함 채택·승격)
│
└── [시스템관리]
    └── [AI 관리]  ← 그룹 헤더 (관리 레이어)
        ├── 에이전트 빌더    #/ai-builder     ← 기존 + 활성 에이전트 카드 띠 추가
        ├── 모니터링         #/ai-monitor     ← 기존 + 6축 도식 + 세션 상세 패널
        ├── 지식베이스       #/ai-kb          ← 기존 유지
        ├── 평가·품질        #/ai-eval        ← 기존 유지
        └── 거버넌스         #/ai-governance  ← 기존 + 역할 3종 권한 매트릭스
```

### Genspark 14개 메뉴 중 우리가 선택하지 않은 것

| 제외 메뉴 | 제외 이유 |
|---------|---------|
| 에이전트 카탈로그 (독립) | 빌더 상단 카드 띠로 흡수. 독립 화면은 데모에서 빈 껍데기 위험 |
| 툴·커넥터 | 외부 연동 운영 기능. 데모에서 설명 부담만 증가 |
| 비용·사용량 (독립) | 토큰 비용은 데모 맥락에 안 맞음. 모니터링 KPI 1카드로 충분 |
| 배포·버전 | CI/CD·롤백은 운영 전용. 데모 임팩트 없음 |
| 자산 허브 | "AI 자산(프롬프트·템플릿)" ≠ "고정자산" — 단어 충돌로 발표 중 혼란 유발 |
| 조직·권한관리 (독립) | 거버넌스 "권한 매트릭스 탭"으로 흡수 |

## 3. 화면별 상세 설계

### 화면 ①: AI Copilot (`#/ai-copilot`)

**목적**: 자산 담당자가 자연어로 자산 현황을 질의하고 즉시 답변을 받는 핵심 사용자 화면.
**개선 방향**: 기존 `#/ai-agent` 3열 채팅 UI 유지 + 아래 3가지 추가.

**[추가 A] 출처 인용 칩 (Citation)**

```
[AI 답변 버블]
서울 지역 유휴 자산은 총 14건, 취득금액 기준 38억원입니다.

근거 자산: [AST-2024-001 ↗] [AST-2024-007 ↗] ... +12건
신뢰도 GOOD  ·  근거 14건  ·  데이터 기준 2026-06-12
```

**[추가 B] 추론 단계 Trace (접힘/펼침 토글)**

```
▶ 추론 과정 보기
  ┌──────────────────────────────────────────────────┐
  │ 1. 자산조회 에이전트   — "서울 유휴 자산" 검색     │
  │ 2. 집계분석 에이전트   — 14건 합산, 금액 계산      │
  │ 3. 답변생성 에이전트   — 결과 포맷·요약            │
  └──────────────────────────────────────────────────┘
```

**[추가 C] 멀티 에이전트 진행 표시** (우리은행 Phase 1 컨셉)

```
처리 중:  ● 자산조회  →  ● 집계분석  →  ○ 답변생성
          (완료)          (진행중)         (대기)
```

**신뢰도 배지**: EXCELLENT(초록) / GOOD(파랑) / MODERATE(주황) / BAD(빨강) — Datumo Eval 컨셉.

**특이 케이스**:
- 가드레일 차단 시 → 차단 유형 메시지 + 차단 배지
- 신뢰도 MODERATE 이하 → "AI 결재함으로 검토 요청" 버튼 노출

### 화면 ②: 이상탐지 보드 (`#/ai-anomaly`)

**목적**: AI가 자산 데이터를 분석해 자동 발견한 위험 항목을 한눈에 확인.

**상단 KPI 바 (4개)**:

| 카드 | 내용 | 강조 색상 |
|------|------|---------|
| 신규 탐지 | 오늘 새로 감지된 위험 자산 수 | 빨강 |
| 위험 자산 합계 | 현재 모니터링 중인 전체 건수 | 주황 |
| 해소 완료 | 이번 주 처리 완료 건 | 초록 |
| 마지막 스캔 | AI 마지막 분석 시각 | 회색 |

**탐지 카드 그리드 (카테고리 필터)**:

```
[필터] 전체 | 노후 고위험 | 장기 유휴 | 보험 만료 임박 | 감가 이상 | 위치 불일치

┌──────────────────┐  ┌──────────────────┐  ┌──────────────────┐
│ ⚠️ 노후 고위험     │  │ 🟡 장기 유휴        │  │ 🔴 보험 만료 D-7   │
│ AST-2019-003     │  │ AST-2021-089     │  │ AST-2023-045     │
│ 서버룸 에어컨      │  │ 회의실 프로젝터    │  │ 법인차 소나타      │
│ 경기 수원         │  │ 서울 여의도        │  │ 부산 해운대        │
│                  │  │                  │  │                  │
│ 내용연수 초과 3년  │  │ 유휴 기간 14개월   │  │ 만료 2026-06-19  │
│ [상세 보기]       │  │ [처리 요청]        │  │ [갱신 요청]       │
└──────────────────┘  └──────────────────┘  └──────────────────┘
```

**하단 탐지 타임라인**:
- 최근 30일 일별 탐지 건수 누적 바 차트 (Chart.js)
- 탐지 이력 테이블: 일자 / 유형 / 자산명 / 상태 / 처리 담당자

**오탐 신고 버튼**: "이건 정상입니다" → AI 학습 피드백 (시뮬레이션)

### 화면 ③: 자동 리포트 (`#/ai-report`)

**목적**: AI가 자산 데이터 기반으로 정기 보고서 초안 자동 생성 → 담당자가 검토·확정 후 AI 결재함으로 전달.

**레이아웃**: 좌측 리포트 유형 선택 / 우측 생성 결과 뷰어

**리포트 유형**:

| 유형 | 주기 | 내용 |
|------|------|------|
| 월간 자산 현황 | 매월 | 증가/감소 자산, 유지보수 현황, 부서별 분포 |
| 분기 감가상각 | 분기 | 감가 현황, 장부가액 변동, 예산 소진율 |
| 연간 자산 실태 | 연 1회 | 전체 자산 목록, 노후 위험도, 교체 권고 목록 |
| 맞춤 리포트 | 요청 시 | 조건 직접 설정 |

**생성 플로우**:
```
① 유형 선택 → ② 기간·범위 설정 → ③ AI 초안 생성 (멀티 에이전트 진행 표시)
→ ④ 결과 뷰어(마크다운 렌더링) → ⑤ 수정 → ⑥ "결재 요청" 버튼 → AI 결재함으로 이동
```

**뷰어 액션 바**: `[AI 초안 재생성]` `[수정]` `[결재 요청 →]` `[임시저장]`

**생성 이력 테이블**:

| 생성일 | 유형 | 기간 | 생성자 | 결재 상태 | |
|--------|------|------|--------|---------|---|

| 2026-05-03 | 월간 현황 | 2026-04 | 이담당 | ⏳ 결재 대기 | 보기 |
| 2026-04-02 | 월간 현황 | 2026-03 | 김담당 | ✅ 승인 완료 | 보기 |

**블록 팔레트 (카테고리별)**:

```

  □ 자산 조회    □ 결과 출력
  □ 사용자 입력  □ 알림 전송

AI 처리
  □ LLM 분석    □ RAG 검색
  □ 요약 생성    □ 신뢰도 채점

제어 흐름
  □ 조건 분기    □ 반복
  □ 승인 게이트  □ 타이머   ← Genspark 승인 게이트 추가

  □ 문서 생성
```

## 6. UI 설계 원칙

### 원칙 1. 개발자용 화면과 현업용 화면을 섞지 않는다 (Genspark 채택)

- **현업 담당자**: AI Copilot, 이상탐지, 자동리포트, AI 결재함 — 수치보다 인사이트와 행동 버튼 중심.
- **AI 운영자**: 빌더, 모니터링, 지식베이스 — 기술 지표와 세부 설정 중심.
- 두 그룹이 섞이면 아무도 만족하지 못한다.

### 원칙 2. KPI는 항상 드릴다운 가능하게 (Genspark 채택)

```
전체 오류율 → 에이전트별 오류율 → 세션 목록 → 세션 상세 타임라인
```
숫자를 클릭하면 그 원인까지 30초 안에 보여야 한다.

### 원칙 3. 운영 이슈의 대부분은 "설정"에서 온다 (Genspark 채택)

모델 자체보다 프롬프트·지식베이스·승인정책·라우팅에서 대부분의 문제가 발생한다. 따라서 각 화면에서 해당 설정으로 바로 이동하는 링크를 항상 제공한다.

### 원칙 4. 대화 보기보다 작업 흐름 보기 우선 (Genspark 채택)

NH 고정자산관리는 채팅 서비스가 아니라 **업무 프로세스 자동화 도구**다. 단순 대화 내역이 아닌 "어떤 업무가 어떤 단계를 거쳐 처리됐는가"를 중심으로 표시한다.

### 원칙 5. 프로토타입 구현 제약 (우리 초안 유지)

- 실제 LLM 연동 없이 정적 목업 데이터로 화면 패턴만 구현.
- 멀티 에이전트 진행·세션 타임라인은 `setTimeout` + CSS 애니메이션으로 시뮬레이션.
- 이상탐지 결과는 `js/data.js` 자산 데이터 기반 사전 계산.
- 추가 라이브러리 불필요 — 기존 Chart.js + Leaflet으로 충분.

## 7. Genspark 제안 반영 판단 근거

### 채택한 것 (이유 포함)

| Genspark 제안 | 채택 방식 | 채택 이유 |
|-------------|---------|---------|
| 6축 프레임워크 | 모니터링 최상단 도식 1장 | 발표 도입부 스토리 장악, 전체 구조 설명에 효과적 |
| 3종 사용자 페르소나 | 거버넌스 권한 매트릭스 3행 + 진입 역할 배지 | 금융권 RBAC 개념 시각화, 설득력 |
| 승인함(Human-in-the-loop) | AI 결재함 독립 화면으로 승격 | "AI가 혼자 결정 안 함" = 금융권 신뢰의 핵심 |
| 세션 트레이스 독립 개념 | 모니터링 "세션 상세 패널"로 간소화 흡수 | 설명가능성 강화. 독립 화면은 데모에 과함 |
| 에이전트 카탈로그 개념 | 빌더 상단 활성 에이전트 카드 띠 | "AI가 팀으로 협업" 인상. 독립 화면은 빈 껍데기 위험 |
| 운영 플로우 3가지 | 화면 간 흐름 섹션 | 발표 시나리오 가이드로 활용 |
| UI 원칙 4가지 | 그대로 채택 | 실무 검증된 내용, 우리 초안과 방향 일치 |

### 채택하지 않은 것 (이유 포함)

| Genspark 제안 | 제외 이유 |
|-------------|---------|
| 배포·버전 관리 | CI/CD·롤백은 운영 전용. 데모 임팩트 없고 콘텐츠 채울 수 없음 |
| 비용·사용량 (독립) | 토큰 단가·예산 이야기는 기획 데모에 불필요. 모니터링 KPI 1카드로 충분 |
| 툴·커넥터 화면 | 외부 연동 운영 기능. 데모 설명 부담만 증가 |
| 자산 허브 | "AI 자산(프롬프트·모델)" ≠ "고정자산" — 핵심 도메인 용어 충돌, 발표 중 혼란 |
| 조직·권한관리 (독립) | 거버넌스 탭 2로 흡수. 독립 화면은 내용 중복 |
| Phase 3 자동 개선·클러스터링 | 데모 프로토타입 범위 초과 |
| 8종 세분화 역할 | Super Admin/Security Admin 등은 데모에서 설명만 길어짐. 3종으로 단순화 |
